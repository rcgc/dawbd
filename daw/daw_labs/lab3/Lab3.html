URL1: son las siglas del acrónimo en inglés "Uniform Resource Locator", que en español significa "Localizador Uniforme de Recursos" y es usada para nombrar recursos en internet. Esto es posible ya que la URL se encuentra compuesta de cierta información que será usada por el navegador para halla el recurso solicitado.

Por otro lado, las direcciones URL nos útiles para evitar memorizar la dirección IP del servidor en que se halla albergado el recurso buscado, por ejemplo, en ligar de escribir 130.0.13.33, solo escribiríamos www.recursosbuscados.com y listo, ahora bien el trabajo de hacer la conversión de una URL a una dirección IP la realiza el DNS "Domain Name Server" (Servidor de Nombres de Dominio).

Los componentes más comunes de los que se encuentra compuesta una URL serán explicados a continuación a partir del siguiente ejemplo:


esquema://host.dominio:puerto/ruta/documento


Esquema: sistema de clasificación que se utiliza para definir el tipo de modo en que los datos y recursos contenidos en la dirección deben ser recuperados, es decir el protocolo que deberá usarse. Ejemplo de ello son: http, https, ftp, mailto, file, news, telnet, entre otros.

Host: ordenador donde se halla albergado el recurso, para acceder a ello empleamos "www".

Dominio: el nombre que se asociará con la IP.

Ruta: la ruta para llegar el directorio.

Archivo: el documento al que queremos acceder; por lo general tienen extensiones .html, .php, .asp, entre otros.



MIME2: son las siglas del acrónimo en inglés "Multipurpose Internet Mail Extensions", que en español significa "Extensiones multipropósito de Correo de Internet" que es una forma estandarizada de indicar la naturaleza y el formato de un documento.

Los navegadores a menudo usan el tipo MIME (y no la extensión de archivo) para determinar cómo procesará un documento; por lo tanto, es importante que los servidores estén configurados correctamente para adjuntar el tipo MIME correcto al encabezado del objeto de respuesta.

Tipos MIME importantes para desarrolladores Web:


application/octet-stream: Este es el valor predeterminado para un archivo binario. Como realmente significa un archivo binario desconocido, los navegadores generalmente no lo ejecutan automáticamente, o incluso preguntan si debería ejecutarse.

text/plain: Este es el valor predeterminado para los archivos de texto. Incluso si realmente significa un archivo textual desconocido, los navegadores asumen que pueden mostrarlo.

text/css: Todos los archivos CSS que deban interpretarse como tales en una página web deben ser de los archivos de text/css. A menudo, los servidores no reconocen los archivos con el sufijo .css como archivos CSS y los envían con tipo MIME text/plain o application/octet-stream: en estos casos, la mayoría de los navegadores no los reconocerán como archivos CSS y serán ignorado silenciosamente.

text/html: Todo el contenido HTML debe ser servido con este tipo. Los tipos MIME alternativos para XHTML, como application/xml+html, son en su mayoría inútiles hoy en día (HTML5 unificó estos formatos).

Tecnologías para diseño web Descripción
XML4

Es un meta-lenguaje que nos permite definir lenguajes de marcado adecuados a usos determinados
XHTML5

XHTML significa lenguaje de marcado de hipertexto extendido. Se utiliza para generar documentos y contenidos de hipertexto generalmente publicados en la WEB. Es además una reformulación del lenguaje HTML que se puede jactar de ser ahora compatible con XML

HTML5 (HyperText Markup Language, versión 5) es la quinta versión del lenguaje de marcado HTML que nace de las necesidades de la Web conforme se va acercando más a los usuarios

Javascript es un lenguaje de programación con muchas posibilidades, utilizado para crear pequeños programas que luego son insertados en una página web y en programas más grandes, orientados a objetos mucho más complejos. Con Javascript podemos crear diferentes efectos e interactuar con nuestros usuarios
PHP8

PHP (acrónimo recursivo de PHP: Hypertext Preprocessor) es un lenguaje de código abierto muy popular especialmente adecuado para el desarrollo web y que puede ser incrustado en HTML
JSP9

JSP es un acrónimo de Java Server Pages, que en castellano vendría a decir algo como Páginas de Servidor Java. Es, pues, una tecnología orientada a crear páginas web con programación en Java
AJAX10

Ajax (Asynchronous JavaScript and XML) se refiere a un grupo de tecnologías que se utilizan para desarrollar aplicaciones web. Al combinar estas tecnologías, las páginas web parece que son más receptivas puesto que los paquetes pequeños de datos se intercambian con el servidor y las páginas web no se vuelven a cargar cada vez que un usuario realiza un cambio de entrada. Ajax permite que un usuario de la aplicación web interactúe con una página web sin la interrupción que implica volver a cargar la página web. La interacción del sitio web ocurre rápidamente sólo con partes de la página de recarga y renovación.


Preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?11

Internet es una red mundial descentralizada de computadoras y servidores, que utilizan la familia de protocolos TCP/IP para intercambiar archivos.

Mientras que la World Wide Web establece un sistema de distribución de documentos de hipertexto (páginas web) que pueden contener textos, imágenes, videos u otros contenidos multimedia ya que dichas páginas están interconectadas entre sí a través de hiperenlaces (links).

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?12

GET: para leer una representación de un recurso
POST: enviar datos
PUT: utilizado normalmente para actualizar contenidos, pero también pueden crearlos.
DELETE: para eliminar un recurso
HEAD: es idéntido a GET, solo el servidor devuelve el contenido de partes concretas del hiperdocumento

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?13

POST, debido a que solamente quieres enviar datos al servidor.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?14

GET

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?15

Que ha habido una respuesta correcta a una petición, por lo tanto no ha ocurrido ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?16

Depende del caso, es error del lado del desarrollador si no estableció bien la ruta al recurso, sin embargo, si es un error a nivel infraestructura la responsabilidad ya no recae del lado del desarrollador.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?17

No, ya que ese error es del lado del servidor, por lo tanto es responsabilidad del que gestiona la infraestructura.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados18.

Significa que dichos atributos serán reemplazados en pro de otros elementos de modo que esos elementos desaprobados ya no tendrán soporte a futuro, ejemplo de ello son: basefont (base font change), center (centered block), font (font change), s (strike-through text), strike (strike-through text), u (underlined text), entre otros.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?19

Las especificaciones de HTML 5 estarán divididas en 2 públicos de destino: los programadores y los dispositivos que lo interpretan (navegadores y demás). Mientras que a los desarrolladores nos presentarán unos requisitos estrictos, que tendrán diversos cambios, a los dispositivos que implementen HTML 5 se les presentarán unas especificaciones más amplias, que respeten la compatibilidad hacia atrás (para que sigan procesándose correctamente las páginas en HTML 4 y anteriores).

HTML sigue siendo un lenguaje de etiquetado, pero ahora la sintaxis puede expresarse en etiquetas como las que venimos utilizando desde siempre y en sintaxis XML, que varía un poquito. Nosotros podremos elegir entre expresarlo en HTML como siempre o en una sintaxis más rígida en XML.

Se crearán diversos elementos nuevos, como SECTION, ARTICLE, ASIDE, HEADER, FOOTER, NAV, DIALOG... incluso habrá dos nuevas etiquetas para insertar AUDIO y VIDEO y otras para hacer cosas tan interesantes como DATAGRID, PROGRESS, TIME, CANVAS, DETAILS...

Otra cosa que llama la atención es el campo INPUT, que ahora tendrá muchos más valores posibles en el atributo type: datetime, date, month, week, number, url, email, color...

¿Qué componentes de estructura y estilo tiene una tabla?20

Secciones: table header (tag theader), table body (tag tbody) y table foot (tag tfoot).

Componentes de estructura: filas (tag tr) y celdas (tag td); las celdas que pertenecen al encabezado usan la tag th.

Componentes de estilo: anchura (width), alineación de texto (texto-align y vertical-align), bordes (border), relleno (padding), ubicación del caption (caption-side), fondo (background), entre otros.

¿Cuáles son los principales controles de una forma HTML5?21

Cajas de texto, cajas de texto para claves, botones de elección, cajas de selección, imágenes, botón de borrado, botón genérico y campos ocultos, entrada de números, barra de rango, calendario y horas, color picker, search text field, datalist, campos de entrada para teléfonos, email y url,

Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?23

De acuerdo al modelo en cascada, el ciclo de vida consta de los siguientes elementos: Definición de requerimientos, Análisis y diseño del software, Implementación y prueba de unidades, Integración y prueba del sistema y finalmente, Operación y mantenimiento.

Ciclo de desarrollo de sistemas de información: Investigación preliminar, Análisis de la información, Diseño del nuevo sistema, Desarrollo y documentación del nuevo sistema, Implantación del sistema de información y finalmente, mantenimiento del sistema.

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?24

Es de suma prioridad para establecer un formato, estándares, entre otros, en un sitio web. Así como también lo será al momento de trabajar con distintos navegadores web.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?25

La resolución, evitar que distraiga al usuario, evitar que dañe la vista, evitar que se irrelevante al contexto de la página web, entre otros.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?26

Habrá casos en los que será importante el porcentaje para que se mantenga la armonia del diseño, proporciones, etc. Por otro lado, habrá veces que la cantidad de puntos y pixeles importe más debido a que un elemento deberá que ser mayor o igual a una medida en pixeles o puntos, ejemplo de ello, un sitio web, una app web que se pueda visualizar en dispositivos móviles.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?27

Debido a que se suprimen los saltos de línea y es más rápida su interpretación y proceso de carga.
