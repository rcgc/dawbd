Laboratorio 7

¿Qué es Materialize?

"Es un framework de frontend desarrollado por 4 estudiantes de Carnegie Mellon University bajo licencia del MIT"

¿Qué es un framework?

"Podemos definir a un framework como un esquema (un esqueleto, un patrón) para el desarrollo y/o la implementación de una aplicación."

"La importancia de utilizar un framework de frontend recae en el hecho de que este nos proveerá de herramientas elaboradas por terceros para modelar una interfaz al usuario. Esto implicará un ahorro de tiempo y con ello centrarnos en las funciones de gestión de información que provee el sitio para resolver las necesidades del cliente (backend)"

Historia

¿Cómo empezó todo?

"Material design es un concepto, una filosofía, un lenguaje de diseño creado por Google con pautas enfocadas al diseño utilizado en la UI de Android, pero también para la web y en cualquier plataforma, se trata de un diseño más limpio en el que predominan animaciones y transiciones de respuesta, el relleno y los efectos de profundidad tales como la iluminación y las sombras."

"MaterializeCSS es la versión estándar que viene con ambos archivos CSS y JavaScript minificados y desminificados. Esta opción requiere poca o ninguna configuración, es la versión ideal para producción."

Características

Framework de frontend

Propicia el diseño de interfaces minimalista

Documentación fácil de navegar con una vasta cantidad de ejemplos

No requiere JQuery, sino que importa su propia hoja de estilos (archivo .css)

Sólo es necesario incorporar dos líneas de código para comenzar a usarlo (importar archivo .css y .js)

Novedades.

Materialize v1.0.0

Ya no hay dependencia de JQuery

Mayor claridad en la documentación y ejemplos

Datepicker and Timepicker han sido rediseñados y reescritos

Se ha mantenido la compatibilidad con JQuery para su uso independiente
